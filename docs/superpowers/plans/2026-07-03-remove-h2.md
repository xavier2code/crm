# Remove H2 Support / PostgreSQL-Only Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove H2 in-memory database support from the backend so the app runs on PostgreSQL-only SQL syntax; local dev connects to localhost PostgreSQL and tests run via Testcontainers.

**Architecture:** Eliminate H2 from dependencies and configuration, switch the default `dev` profile to PostgreSQL, introduce a Testcontainers PostgreSQL base class for integration tests, and clean up H2-specific comments/workarounds in Flyway migrations.

**Tech Stack:** Spring Boot 3.2, Gradle, PostgreSQL 15, Flyway, Testcontainers, JUnit 5.

## Global Constraints

- Work in an isolated git worktree on its own branch (CLAUDE.md requirement).
- One commit per task area using Conventional Commits.
- Backend tests must pass: `cd backend && ./gradlew test`.
- Frontend must still pass: `cd frontend && npm run lint && npm run build` (no frontend changes expected).
- Never edit already-shipped Flyway scripts except to fix PostgreSQL syntax errors; prefer new scripts for behavioral changes.
- Match existing Java style (4-space indent, Lombok).

**Files:**

**Interfaces:**
- Consumes: `org.testcontainers:postgresql`, `org.testcontainers:junit-jupiter`.
- Produces: A reusable `PostgreSQLContainer` and a base class that injects dynamic datasource properties into `@SpringBootTest`.

- [ ] **Step 1: Create shared PostgreSQL container holder**

```java
package com.cy.crm.config;

import org.testcontainers.containers.PostgreSQLContainer;

public class PostgresTestContainer {

    private static final PostgreSQLContainer<?> INSTANCE =
            new PostgreSQLContainer<>("postgres:15-alpine")
                    .withDatabaseName("crm_test")
                    .withUsername("crm")
                    .withPassword("crm")
                    .withReuse(true);

    static {
        INSTANCE.start();
    }

    public static PostgreSQLContainer<?> getInstance() {
        return INSTANCE;
    }

    private PostgresTestContainer() {
    }
}
```

- [ ] **Step 2: Create abstract base test class**

```java
package com.cy.crm.config;

import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.context.DynamicPropertyRegistry;
import org.springframework.test.context.DynamicPropertySource;
import org.testcontainers.junit.jupiter.Testcontainers;

@SpringBootTest
@Testcontainers
public abstract class AbstractPostgresIntegrationTest {

    @DynamicPropertySource
    static void registerDatasourceProperties(DynamicPropertyRegistry registry) {
        PostgreSQLContainer<?> postgres = PostgresTestContainer.getInstance();
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
        registry.add("spring.datasource.username", postgres::getUsername);
        registry.add("spring.datasource.password", postgres::getPassword);
        registry.add("spring.datasource.driver-class-name", postgres::getDriverClassName);
    }
}
```
